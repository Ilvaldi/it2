Grid (repetisjon fra IT1)

Å strukturere innhold vertikalt

Div-elementer er fine for å strukturere en side. Div-elementet er et blockelement, dvs at nye elementer automatisk legger seg under div-elementet. Ved å overstyre dette med css kan vi få en nettside med innhold plassert ved siden av hverandre.

For at plasseringene skal holde seg likt når på skjermer av ulike dimensjoner og når vi endrer størrelse på nettleservinduet, bruker vi enten flexboxer eller grid der vi angir alle størrelser i prosent av moderelementet. Det finnes flere utgangspunkter for et grid, disse notatene baserer seg på dette.

Grid

Vi kommer til å forholde oss til grid for å strukturere innhold. Gridet er en slags tabell der hver rad har ulikt antall kolonner og ulik størrelse på kolonnene.

Alt innhold kan vi da legge i disse div'ene. For at det skal leses som grid må følgende kode inngå i css:

  * { /* gjelder alle html-elementer */
    box-sizing: border-box;
  }
  [class*="col-"] { /* gjelder alle klasser som heter col- */
    float: left;
    padding: 15px; /*vi setter selv en passende padding */
    border: 1px solid red; /* kun for å se boksene visuelt */
  }
    .row::after { /*innholdet etter en rad */
    content: "";
    clear: both;
    display: table;
  }

I tillegg definerer vi de breddene vi tror vi kan få bruk for. Følgende oppsett er et godt forslag, da hver rad vil summeres til 100% så lenge kolonnenumrene summeres til 12:
  .col-1 {width: 8.33%;}
  .col-2 {width: 16.66%;}
  .col-3 {width: 25%;}
  .col-4 {width: 33.33%;}
  .col-5 {width: 41.66%;}
  .col-6 {width: 50%;}
  .col-7 {width: 58.33%;}
  .col-8 {width: 66.66%;}
  .col-9 {width: 75%;}
  .col-10 {width: 83.33%;}
  .col-11 {width: 91.66%;}
  .col-12 {width: 100%;}

Man kan selvfølgelig også lage seg sine egne standardstørrelser, det eneste man må passe på er at bredden summerer seg til 100%.

Videre muligheter med grid

Vi kan for eksempel en lage en meny ved hjelp av div-bokser som kun inneholder en lenke. Dersom man har tre lenker kan man se for seg en rad som består av:
   16.66 % tom div
   16.66 % lenkediv
   8.33 % tom div
   16.66 % lenkediv
   8.33 % tom div
   16.66 % lenkediv
   16.66 % tom div

lenke 1
lenke 2
lenke 3

HTML-kode for denne (legg merke til at < a> ligger før < div>), det gjør at hele boksen utgjør lenken:
   < div class="row lenker">
     < div class="col-2"> < /div>
     < a href=""> < div class="col-2 lenke"> lenke 1 < /div>< /a>
     < div class="col-1">
     < a href="">< div class="col-2 lenke"> lenke 2 < /div>< /a>
     < div class="col-1"> /div>
     < a href="">< div class="col-2 lenke"> lenke 3 < /div>< /a>
     < div class="col-2"> < /div>
   < /div>

CSS-kode for denne, i tillegg til grid-css:
   .lenker div{
     border: none; /* tar bort den røde kantlinja */
   }
   .lenke{
     background-color: #8bb1ad;
     border-radius: 10px;
     text-align: center;
   }
   .lenke a{
     text-decoration: none;
   }

Som et eksempel kan en side bestå av:

  • En omkringliggende < div class = "col-12 ramme"> < /div > som rammer inn alt innhold på siden og har bredde 100 %
  • En header < div class ="col-12 header"> < /div > som inneholder en overskrift med et bakgrunnsbilde
  • Lenker i div-elementer
  • To elementer ved siden av hverandre der det ene har < div class = "col-9" > < /div > og det andre har < div class = "col-3"> < /div > og begge inneholder hvert sitt bilde
  • Et canvaselement som er midtstilt på siden med en tom < div class = "col-2" > < /div > på hver side

Oppgave

Lag siden beskrevet over ved bruk av grid. Gi elementene ulike skriftfarger/bakgrunnsfarger/kantlinjer slik at du ser dem tydelig. Prøv deg frem videre med å legge andre elementer i klasser med andre størrelser og se hva som skjer dersom kolonnebreddene ikke summerer seg til 100%.
