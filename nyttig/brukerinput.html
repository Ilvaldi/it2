Vi repeterer hvordan vi legger til skjemaer og henter inn informasjon med javascript

HTML for ulike inputfelt

I IT-1 la dere inn alle skjemaer i taggen < form > og la inn en metode (post eller get) for å hente inn informasjon med php. Disse skjemaene måtte ha en knapp i form av < input type="submit" > for å kunne sende informasjon. Med JavaScript er det ikke nødvendig med < form >-taggen for annet enn stilsetting, og når vi bruker hendelsesstyrte funksjoner kan vi sende informasjonen i et skjema med alle slags hendelser.

Vi har fortsatt bruk for å legge inn et skjema med inputfelt av ulike typer, og her vises de vanligste typene:

Inputfelt Beskrivelse
< input type = "text" > Gir et textfelt. Datatypen til det som skrives inn her er string.
< input type = "number" > Datatypen til det som skrives inn her er string, men kan gjøres om til number.
< select > Gir en valgmuligheter som dropdown
< input type = "radio" > Gir valgmuligheter som radioknapper (kan huke av én)
< input type = "checkbox" > Gir valgmuligheter som checkboxer (kan huke av flere)
< input type = "submit" > Gir en "send-knapp". Denne er ikke nødvendig for at skjemaet skal fungere.
< option > Hver valgmulighet i en select legges i en < option >
For de ulike input feltene er det flere ting vi må huske på.
  • Alle inputfelt må ha et navn, som settes som attributt "name = navn". Uten dette kan vi ikke identifisere hva slags verdi som legges inn.
  • Number-felt kan få en min- og max-verdi ved å sette som attributt min = "0" max = "10". Det går fortsatt an å skrive inn andre verdier manuelt, så disse attributtener erstatter ikke en sjekk av at brukeren har skrevet inn gyldig verdi.
  • Radioknapper til samme felt må ha samme verdi for name. Dette for at det ikke skal være mulig å knysse av for flere radioknapper. Det samme bør checkboxer til samme felt ha.
  • Radioknapper og checkboxer må også få en unik value eller id sånn at man kan finne ut av hvilke(n) som er valgt
  • Vi kan legge enn en placeholder i et felt med attributtet placeholder = "tekst som vises"
Koden for et skjema kan dermed være

   < form > // denne taggen er ikke nødvendig
     Skriv inn navn: < input type = "text" name = "navn" > < br >
     Skriv inn alder: < input type = "number" name = "alder" min = "0" max = "100"> < br >
     Hva synes du om denne nettsiden? < select name="tilbakemelding" >
       < option value="1" > Dårlig < /option >
       < option value="2" > Helt OK < /option >
       < option value="3" > Kjempebra! < /option >
     < /select > < br >
     Kjønn: < input type = "radio" name = "kjonn" value = "k"> Kvinne < input type="radio" name="kjonn" value = "m"> Mann < br >
     Hva liker du å spise til middag?
     < input type="checkbox" name="middag" value="pasta"> Pasta < br >
     < input type="checkbox" name="middag" value="pizza"> Pizza < br>
     < input type="checkbox" name="middag" value="taco"> Taco < br >
     < input type="submit" name="send" value="Send tilbakemelding" > // her kunne det også vært en div, en button eller hva som helst annet
   < /form > // denne taggen er ikke nødvendig

Stilsetting av skjemaer

Alle inputelementer er inlineelementer. Vi kan overstyre i html ved å sette inn < br > etter hvert element, eller vi kan gå inn i css og bestemme at
   input, select{
     display: block;
   }

Når vi ellers stilsetter skjemaet, har vi valget mellom å stilsettte
   form {}/* stilsetter hele skjemaet gitt at det ligger i form-tagger (det kommer det ikke til å gjøre) */
   input[type="text"]{} /* stilsetter alle inputfelt av typen text
   input[type="number"]{} /* stilsetter alle inputfelt av typen number
   input[type="submit"]{} /* stilsettter alle inputfelt av typen submit
   /* osv for andre inputfelt */
I tillegg kan enkeltfelt stilsettes ved bruk av klasser eller id'er.

Hendelser for å hente inn informasjon

Som oftest bruker vi en klikkhendelse der lytteren er en knapp for å sende inn all informasjon fra et skjema samlet. Det kan også være aktuelt å legge hendelsen change til select-elementer direkte:
   < select name = "test" id="test" >
     < option value="1" >1< /option >
     < option value="2" >2< /option >
   < /select >
   < p id="skrivHer" >< /p >
I scriptet henter man ut verdien til "test" med en gang:
   var testEl=document.querySelector("#test");
   testEl.addEventListener("change", skrivUt);
   var skrivHerEl = document.querySelector("#skrivHer");
   function skrivUt(){
     skrivHerEl.innerHTML="du har valgt "+testEl.value;
   }

Eksempel: henter inn info i skjemaet over

I skjemaet over har ga inputfeltene hver sin id. Dette er for senere å kunne hente ut svarene. Disse svarene kommer jeg til å skrive ut til et tomt div-element jeg har lagt under skjemaet. Jeg ønsker ikke å hente ut svarene før knappen trykkes, og legger derfor til en lytter:
   var sendEl = document.querySelector("#send");
   sendEl.addEventListener("click", sendSvar);

Jeg henter også ut kvitteringsdiven som jeg skal skrive til:
   var kvitteringEl = document.querySelector("#kvittering");

I funksjonen sendSvar vil jeg finne ut hva som er registrert i feltene og skriver ut kvittering:
   function sendSvar(){
     // henter inn svar
     var navnEl = document.querySelector("#navn");
     var tlfEl = document.querySelector("#tlf");
     var gjesterEl = document.querySelector("#ant");
     var frokostJaEl = document.querySelector("#frokostJa");
     var frokostNeiEl = document.querySelector("#frokostNei");
     var aktBoEl = document.querySelector("#aktBo");
     var aktHoytEl = document.querySelector("#aktHoyt");
     var aktFruktEl = document.querySelector("#aktFrukt");
     var aktTelemarkEl = document.querySelector("#aktTelemark");

     // finner ut om gjestene skal ha frokost
     var frokost ="";
     if(frokostJaEl.checked){
       frokost = "skal ha frokost";
     }
     else if(frokostNeiEl.checked){
       frokost = "skal ikke ha frokost";
     }

     // finner ut hvilke aktiviteter
     var aktBo = "";
     if(aktBoEl.checked){
       aktBo = "Bø Sommarland < br >";
     }
     var aktHoyt = "";
     if(aktHoytEl.checked){
       aktHoyt = "Høyt og Lavt klatrepark< br >";
     }
     var aktFrukt = "";
     if(aktFruktEl.checked){
       aktFrukt = "Omvisning på fruktgård< br >";
     }
     var aktTelemark = "";
     if(aktTelemarkEl.checked){
       aktTelemark = "Dagstur på Telemarkskanalen< br >";
     }
     var aktIngen = "";
     if(!(aktBoEl.checked || aktHoytEl.checked || aktFruktEl.checked || aktTelemarkEl.checked)){
       var aktIngen = "Ingen aktiviteter < br >";
     }

     fyller tom div med kvittering
     kvitteringEl.innerHTML = "< p > Navnet er " + navnEl.value + " og du er registrert med telefonnr " + tlfEl.value + ". Dere er " + gjesterEl.value + " gjester, og dere " + frokost + ". Av aktiviteter har dere valgt: < br >" + aktBo + aktHoyt + aktFrukt + aktTelemark + aktIngen + "< /p >";
   }

Bestillingsverktøy for matkasse
Bestillingen av matkassene skal foregå på nettsiden. Du skal lage en applikasjon der kunden (brukeren) kan velge en matkasse som inneholder to eller tre middager, og hvor mange personer middagene skal lages til.
Når brukeren har valgt matkasse, skal det komme opp et bilde som illustrerer om det er to eller tre middager som er valgt. Applikasjonen bør sjekke om antall personer som er registrert, er innenfor lovlige/sannsynlige verdier.
Prisen beregnes ut fra valgene, og du kan regne med 80 kroner per person per måltid. Bestilles det matkasser til fem personer eller mer, vil prisen per person være 70 kroner.
Applikasjonen skal bekrefte valgene som er gjort, og navnet og adressen til den som bestiller, skal legges inn og vises sammen med de andre opplysningene.
(Bestillingen vil på en virkelig nettside måtte sendes/lagres ved hjelp av for eksempel epost eller en fil, men på eksamen trenger du ikke gjøre noe med dette.)
